YouTube Music MCP

Control YouTube Music from MCP clients through the Model Context Protocol. Install the macOS bridge app from the DMG, enable the Safari extension, and connect Codex, Claude Code, Claude Desktop, Gemini CLI, GitHub Copilot, Cursor, Windsurf, VS Code, or another MCP-compatible client.

How It Works

A request starts in your MCP client, moves through the local bridge stack, and ends in the active YouTube Music tab.

1 MCP Client

Codex, Claude Code, Claude Desktop, Gemini CLI, GitHub Copilot, Cursor, Windsurf, VS Code, or another MCP-compatible client can invoke the tools.

MCP / JSON-RPC
2 Go MCP Server

The local server exposes the MCP tool surface and turns client requests into bridge commands.

HTTP / localhost:8765
3 macOS Bridge App

The menu bar app keeps the local HTTP endpoint alive and coordinates the live Safari connection state.

WebSocket
4 Safari Extension

The extension runs inside Safari, forwards actions to the page, and reads playback state back out.

DOM automation
5 YouTube Music

The final action happens in the actual YouTube Music web player running in your active tab.

Quick Start

1. Install the bridge app from the DMG

Download the latest DMG, drag "YouTube Music MCP Bridge.app" to Applications, and open the app once.

2. Enable the Safari extension

In Safari, go to Settings > Extensions, enable "YouTube Music MCP Bridge", then open music.youtube.com.

3. Follow the setup guide for your client

The setup guide includes client-specific setup for Codex, Claude Code, Claude Desktop, Cursor, Gemini CLI, GitHub Copilot, VS Code, and Windsurf.

4. Ask your client to play music

> Play some lo-fi hip hop on YouTube Music
